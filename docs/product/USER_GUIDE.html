01

로그인

영업팀 계정으로 로그인하면 통화 어시스턴트로 진입합니다. 멀티 조직 사용자는 로그인 시 조직 선택 안내가 자동으로 노출됩니다.

✓ 현재 가능
  • 이메일 + 비밀번호 로그인 (Argon2id 해시)
  • HttpOnly refresh cookie 30일 + Bearer access 15분 갱신
  • 잘못된 자격증명·다중 조직 시 인라인 안내 메시지
  • localhost 접속 시 dev 시드 4쌍 + 자동 채우기 버튼 노출
  • ✓ Phase 3에서 추가: 회원가입 UI + 24시간 이메일 인증 메일
  • ✓ Phase 3에서 추가: 비밀번호 분실 → 메일 링크로 재설정 (적용 시 모든 활성 세션 revoke)
  • ✓ Phase 3에서 추가: 멀티-조직 계정은 1차 로그인 실패 응답에 organization 리스트가 포함되어 조직 ID 입력 안내 노출
⌛ 아직 안 됨
  • SSO / SAML — Enterprise 단계 (Phase 6+)
  • 조직 picker UI 1차 화면화 — 현재는 로그인 1차 시도 후 자동 노출 또는 "조직 ID 직접 입력" 토글 (Phase 4+ UX polish)
02

대시보드 — 하루의 시작

로그인 직후 마주하는 메인 화면. 4종 KPI, 시장 트렌드 알림, AI가 생성한 오늘의 To-Do를 한눈에. 어디서부터 시작할지가 자명해집니다.

✓ 현재 가능
  • 오늘 통화 / 응답률 / 평균 통화 길이 / 진행 중인 통화 — 4개 카드가 본인 회사 데이터로 채워집니다.
  • 최근 통화 5건이 종료된 시각 순으로 같이 보입니다.
  • 인사말에 본인 이름이 표시되고, 이메일 인증을 아직 안 했다면 상단에 안내 띠가 뜹니다.
  • 알림 패널 · 토스트 · 모달 인터랙션, 사이드바 드로어 전환(좁은 화면)이 정상 동작합니다.
⌛ 아직 안 됨
  • 시장 트렌드 알림 5건 — 네이버 검색 API 연동 (Phase 5~6, 현재 demo)
  • 오늘의 추천 To-Do — AI 추천 엔진 (Phase 5, 현재 demo)
  • 팀 활동 피드 — activity_log 도입 (Phase 5+, 현재 demo)
03

실시간 통화 — 핵심 기능

Kloser의 핵심 통화 화면입니다. 현재는 데모 대화가 실시간처럼 흘러가며 전사, 추천 멘트, 감정 변화가 갱신됩니다. Phase 4에서는 여기서 만든 통화와 메모, 종료 상태가 통화 기록과 대시보드에 남도록 연결됐습니다.

✓ 현재 가능
  • 로그인한 사용자만 통화 화면에 들어갈 수 있습니다.
  • 데모 대화가 실시간으로 흘러가며 전사, 추천 멘트, 감정 변화가 화면에 표시됩니다.
  • 통화 중 빠른 메모를 저장하면 나중에 통화 기록 상세에서 다시 볼 수 있습니다.
  • 종료 버튼을 누르면 통화가 완료 상태로 저장되고 대시보드 최근 통화에도 반영됩니다.
  • 상담 체크리스트와 빠른 응대 칩 UI는 아직 데모 보조 영역입니다.
⌛ 아직 안 됨
  • 실제 STT (Naver Clova) 연동 — Phase 5
  • 실제 LLM 응대 추천 (Claude / RAG) — Phase 5
  • 회사 가이드 / FAQ 벡터 검색 (pgvector) — Phase 5
  • 통화 종료 후 자동 메모 + 다음 액션 분류 — Phase 5
  • 현재 데모는 setTimeout 기반 fixture로 재생됩니다.
04

오늘의 일 — 시장 트렌드 + To-Do

오늘 확인할 시장 이슈와 To-Do를 한 화면에서 보는 데모 화면입니다. 현재는 예시 데이터로 구성되어 있고, 회의 자료를 5포맷(HTML / PDF / Word / Excel / PPT)으로 다운로드하는 흐름을 확인할 수 있습니다.

✓ 현재 가능
  • 모니터링 관심사 chip 추가·삭제 · To-Do 체크박스 토글
  • 5 포맷 다운로드 실작동 — jsPDF / html-docx-js / SheetJS / PptxGenJS
  • Excel은 5개 시트 자동 분리 (요약·트렌드·주간·To-Do·경쟁사)
  • PowerPoint는 6장 자동 구성 (표지·KPI·트렌드·To-Do·경쟁사·마무리)
⌛ 아직 안 됨
  • 실제 네이버 검색 API 호출 — Phase 5~6
  • 실제 To-Do 생성 엔진 (시장 + 통화 결합) — Phase 5
  • To-Do 우선순위 자동 정렬 모델 — Phase 5
05

통화 기록 — 검색 가능한 과거

지난 통화 전부가 검색·필터 가능한 상태로 보존됩니다. 클릭하면 우측에 상세 패널이 슬라이드되어 transcript · 요약 · 다음 액션을 확인.

✓ 현재 가능
  • 실시간 통화에서 종료한 통화가 통화 기록에 그대로 남습니다.
  • 통화 기록 화면에서 검색어와 상태 필터로 통화를 찾을 수 있습니다.
  • 목록에서 통화를 클릭하면 오른쪽 상세 패널에서 메모·대화 내용·액션 아이템을 한 번에 볼 수 있습니다.
  • 다른 회사의 통화 기록은 보이지 않습니다 (회사 단위 분리).
  • 대시보드의 최근 통화에도 종료된 통화가 반영됩니다.
⌛ 아직 안 됨
  • 녹음 음성을 실제로 받아 적어 주는 기능 (실 STT)은 다음 단계입니다.
  • AI가 자동으로 통화 요약과 추천 답변을 만들어 주는 기능은 다음 단계입니다.
  • 액션 아이템을 화면에서 새로 만들거나 완료 처리하는 기능은 아직 없습니다.
  • 고객을 선택해서 통화에 연결하는 흐름은 아직 없습니다.
  • CSV 내보내기 버튼은 아직 실제 다운로드 기능이 아닙니다 (자리만 잡혀 있음).
  • 대화 내용 전체를 본문 검색하는 기능은 아직 없습니다.
06

고객 관리 — 통합 CRM

영업팀의 고객 마스터. 활성·검토중·대기 상태별 필터, 이름·회사·이메일 검색, 마지막 연락 시점까지 한 화면에. Phase 2에서 실 API CRUD로 전환 완료 — 추가·수정·삭제가 영속 저장되고 조직별 격리(RLS)로 다른 회사 고객은 보이지 않습니다.

✓ 현재 가능
  • 실 API CRUD — 추가/수정/삭제 모두 영속 저장 (`/customers` 6 endpoints)
  • 4 KPI (전체/활성/검토중/대기) — 조직 전체 기준 자동 갱신
  • 2 그룹 chip 필터 (status / sort) + 이름·회사·이메일 검색
  • URL query string 동기화 — 새로고침·뒤로 가기·북마크 복원
  • 듀얼 모드 모달 (신규 + 수정 + 삭제) · 24명 seed (Acme 12 / Beta 12)
  • 조직별 격리 — DB RLS로 다른 회사 고객 비노출, viewer 쓰기 403
  • 입력 검증 (zod 서버 + JSDoc 화면) · soft delete (보존)
  • 자동 회귀 — `phase_2_customers_e2e` 7 시나리오 + cleanup sweep
⌛ 아직 안 됨
  • 행 클릭 상세 패널 (통화 이력 결합 UI) — Phase 5 (통화 영속화는 Phase 4에서 완료, customers ↔ calls 결합 화면은 후속)
  • 팀 단위 권한 분기 ("매니저는 자기 팀만") — Phase 5 (현재 admin-only mutation으로 단순화)
  • CSV 일괄 가져오기 — Phase 6+ enterprise
  • HubSpot / Salesforce 양방향 동기화 — Phase 6 (v2 베타)
  • 고급 검색 (오타 허용·전문 검색) — Phase 5+
  • 삭제 고객 복원 UI — Phase 4+ 관리자 콘솔
07

뉴스레터 — AI 챗 기반 작성

"환영 메일 작성해줘" 한 줄로 개인화된 초안 자동 생성. 발송 후엔 전달률·오픈율·클릭률을 실시간 추적합니다.

✓ 현재 가능
  • 캠페인 5개 + 통계 mock
  • AI 챗봇 UI — 4종 프롬프트 (환영/업데이트/웨비나/갱신)별 다른 초안 자동 생성 + 타이핑 점 애니메이션
⌛ 아직 안 됨
  • 실제 LLM 연동 (Claude / GPT) — Phase 5
  • 실제 메일 발송 인프라 (SMTP / SES) — Phase 6+ 운영 진입 단계 어댑터. 현재는 dev outbox 테이블에 메일 본문 보존
  • 발송 통계 실시간 트래킹 — Phase 6
08

팀 & 계정 — 권한 매트릭스

관리자 1개 + 직원 서브 계정 (Pro 1명 / Enterprise 5명). 권한은 Admin · Manager · Employee · Viewer 4단계, 우수 사례 자동 노출로 팀 내 베스트 프랙티스를 즉시 공유.

✓ 현재 가능
  • 구성원 목록 + 권한 매트릭스 UI (실 /team/members API)
  • 직원 초대 모달 — Phase 3에서 실 API로 wiring (보내기·다시 보내기·취소 + 익명 수락 페이지)
  • 서버 측 role guard — admin / manager / employee / viewer 4단계 미들웨어 동작 + 마지막 admin 보호 + requireFreshRole로 stale role 차단
  • ✓ Phase 3에서 추가: 초대 토큰 발급·소비 (sha256 해시 저장, 7일 TTL)
⌛ 아직 안 됨
  • 실제 외부 메일 발송 (SMTP / SES) — Phase 6+ 운영 진입 단계. 현재는 dev outbox 테이블에 메일 본문 보존
  • 팀 CRUD UI (팀 생성·수정·삭제) — Phase 4+ (초대 모달의 팀 드롭다운만 현재 GET /teams 사용)
  • 우수 사례 자동 추출 엔진 — Phase 5
09

설정 — 12 카테고리

프로필부터 위험 영역까지 12개 카테고리. 좌측 스크롤스파이 TOC가 현재 보고 있는 섹션을 항상 표시합니다.

✓ 현재 가능
  • 12 카테고리 — 프로필·회사·통화환경·AI·통합·알림·보안·데이터·플랜·API·언어·위험영역
  • 스크롤스파이 TOC — 보고 있는 섹션 자동 강조
⌛ 아직 안 됨
  • 각 항목 저장 — Phase 4+ (해당 기능 구현 시점에)
  • API 키 발급 — Phase 6
  • 2단계 인증 (TOTP / WebAuthn) — Phase 6+ 운영 진입 단계

Phase 1·2·3·4 종합 정리

지금 시점에서 무엇이 진짜로 동작하고 무엇이 mock인지를 한 표로 정리. Phase 1 기반(인증·격리·권한·세션·실시간) + Phase 2 첫 비즈니스 entity(고객 CRUD) + Phase 3 셀프서비스(가입·인증·복구·초대·팀) + Phase 4 통화 영속화(calls / transcripts / action items + dashboard 실 KPI)까지 완료된 상태.

영역 현재 (Phase 1·2·3·4) 예정
데이터 영속 PostgreSQL 16 + RLS FORCE default-deny — customers (Phase 2) + auth_tokens / email_outbox / 초대 (Phase 3) + calls / transcripts / call_action_items (Phase 4) Phase 5 — knowledge_bases (pgvector) / 통화 녹음 / activity_log
고객 관리 (CRM) 실 API CRUD 동작 — 6 endpoints + 4 KPI + 검색·필터·정렬 (URL 동기화) + soft delete + 24명 seed. 통화 종료 시 customers.last_contacted_at 자동 갱신 Phase 5 — 행 클릭 상세 패널 (통화 이력 결합 UI)
인증 / 셀프서비스 자체 Auth — Argon2id + JWT + refresh rotation + role guard. Phase 3 추가 — 자가 회원가입 / 이메일 인증 / 비밀번호 재설정 / 동료 초대 / 마지막 admin 보호. Phase 4 추가 — 미인증 mutation 차단 (requireVerified) Phase 6+ — SMTP / 2FA / 운영 도메인
조직별 격리 (RLS) DB 차원 강제 — Phase 1 인프라 + Phase 2 customers + Phase 3 invitations/auth_tokens + Phase 4 calls/transcripts/action_items 모두 FORCE RLS + composite FK 새 entity 추가 시 같은 패턴
입력 검증 / 형식 동기화 zod 서버 + JSDoc 화면 + sync 테스트 (9 entity: customers / signup / password-reset / team / invitation / call / transcript / actionItem / dashboard) 새 entity 추가 시 한 줄 등록
통화 어시스턴트 WebSocket (인증 핸드셰이크) + 데모 시퀀스 fixture. Phase 4 추가 — start_call / text_chunk / end_call이 DB 영속 + endCall 트랜잭션이 last_contacted_at 갱신 Phase 5 — 실제 STT (Clova) + LLM (Claude) + RAG + disconnect heartbeat
대시보드 / KPI Phase 4 — `/dashboard/summary` 실 API: 오늘 통화 / 응답률 / 평균 통화 / 진행 중 + 최근 통화 5건. 시장 트렌드 / 추천 To-Do / 팀 활동은 demo (헤더 라벨) Phase 5 — AI 추천 To-Do / 매니저 보고서 / 팀 KPI 차트
시장 트렌드 5 포맷 다운로드 (HTML/PDF/Word/Excel/PPT) 실작동, 데이터는 mock Phase 5~6 — 네이버 검색 API + To-Do 엔진
팀 / 권한 role 미들웨어 4단계 동작 (admin / manager / employee / viewer). Phase 3 추가 — 초대 발송 / 수락 / 멤버 역할·상태 변경 / 마지막 admin 보호. Phase 4 추가 — calls 권한 매트릭스 + employee 본인 통화만 mutation Phase 5 — manager team-scope ("자기 팀 통화만")
자동 회귀 서버 단위 212/212 + sync 9 entity + Phase 0.5 e2e 16/16 + customers e2e 7/7 + Phase 3 e2e 33 assertion + Phase 4 e2e 8 시나리오 + cleanup sweep 새 entity마다 같은 패턴 추가
데스크톱 앱 웹 콘솔만 동작 별도 트랙 — Electron/Tauri + WASAPI 오디오 캡처
