01

로그인

영업팀 계정으로 로그인하면 통화 어시스턴트로 진입합니다. 멀티 조직 사용자는 로그인 시 조직 선택 안내가 자동으로 노출됩니다.

✓ 현재 가능
  • 이메일 + 비밀번호 로그인 (Argon2id 해시)
  • HttpOnly refresh cookie 30일 + Bearer access 15분 갱신
  • 잘못된 자격증명·다중 조직 시 인라인 안내 메시지
  • localhost 접속 시 dev 시드 4쌍 + 자동 채우기 버튼 노출
⌛ 아직 안 됨
  • 회원가입 UI (Phase 3, 이메일 인증과 함께)
  • 비밀번호 재설정 / 마법 링크 (Phase 3)
  • SSO / SAML — Enterprise 단계
  • 조직 picker UI — 현재는 ID 직접 입력 (Phase 3+ 회원가입과 함께 도입 예정)
02

대시보드 — 하루의 시작

로그인 직후 마주하는 메인 화면. 4종 KPI, 시장 트렌드 알림, AI가 생성한 오늘의 To-Do를 한눈에. 어디서부터 시작할지가 자명해집니다.

✓ 현재 가능
  • 4 KPI · 트렌드 위젯 · To-Do 체크박스 · 알림 패널 (UI 인터랙션)
  • fade-in / 토스트 / 모달 애니메이션 — 데모 외관 동등
  • 반응형 — 1024px 미만에서 사이드바가 드로어로 전환
⌛ 아직 안 됨
  • 실제 KPI 집계 — 현재 mock 숫자 (Phase 4 calls REST)
  • 실제 시장 트렌드 수집 — 네이버 검색 API (Phase 4~5)
  • To-Do 자동 생성 엔진 — 통화 기록 + 트렌드 결합 (Phase 5)
03

실시간 통화 — 핵심 기능

Kloser의 차별점. 통화 중 STT가 실시간 전사를 만들고, RAG 기반 AI가 2~3초 안에 응대 멘트를 추천합니다. 우측엔 고객 감정 변화(관심 → 망설임 → 재고려)가 실시간 갱신.

✓ 현재 가능
  • 인증된 WebSocket으로 데모 시퀀스 재생 (Phase 0.5 spike)
  • JWT 핸드셰이크 — 서버가 토큰을 검증해야만 통화 채널 진입
  • transcript / suggestion / sentiment 이벤트 스트리밍
  • RTT 0~5ms 라운드트립 — 클릭에서 화면 갱신까지 체감 즉시
  • 4 KPI · 상담 체크리스트 · 빠른 응대 칩 UI
⌛ 아직 안 됨
  • 실제 STT (Naver Clova) 연동 — Phase 5
  • 실제 LLM 응대 추천 (Claude / RAG) — Phase 5
  • 회사 가이드 / FAQ 벡터 검색 (pgvector) — Phase 5
  • 통화 종료 후 자동 메모 + 다음 액션 분류 — Phase 5
  • 현재 데모는 setTimeout 기반 fixture로 재생됩니다.
04

오늘의 일 — 시장 트렌드 + To-Do

매일 06:00에 자동 갱신. 회사가 지정한 모니터링 키워드와 경쟁사 동향을 네이버 검색 API로 수집해 우선순위 To-Do로 정리합니다. 회의 자료를 5포맷(HTML / PDF / Word / Excel / PPT)으로 즉시 다운로드.

✓ 현재 가능
  • 모니터링 관심사 chip 추가·삭제 · To-Do 체크박스 토글
  • 5 포맷 다운로드 실작동 — jsPDF / html-docx-js / SheetJS / PptxGenJS
  • Excel은 5개 시트 자동 분리 (요약·트렌드·주간·To-Do·경쟁사)
  • PowerPoint는 6장 자동 구성 (표지·KPI·트렌드·To-Do·경쟁사·마무리)
⌛ 아직 안 됨
  • 실제 네이버 검색 API 호출 — Phase 4~5
  • 실제 To-Do 생성 엔진 (시장 + 통화 결합) — Phase 5
  • To-Do 우선순위 자동 정렬 모델 — Phase 5
05

통화 기록 — 검색 가능한 과거

지난 통화 전부가 검색·필터 가능한 상태로 보존됩니다. 클릭하면 우측에 상세 패널이 슬라이드되어 transcript · 요약 · 다음 액션을 확인.

✓ 현재 가능
  • 검색·필터 UI · 8건 mock 통화 데이터
  • 우측 상세 패널 슬라이드 — 클릭 인터랙션
  • 회사 단위 데이터 통합 — 떠난 직원의 통화도 유지
⌛ 아직 안 됨
  • DB 영속 통화 기록 — Phase 4 calls REST + dashboard
  • RLS 기반 조직 격리 — DB 스키마는 Phase 1에 준비됨, REST 라우트는 Phase 4
  • 전문 검색 (transcript 본문) — Phase 4~5
06

고객 관리 — 통합 CRM

영업팀의 고객 마스터. 활성·검토중·대기 상태별 필터, 이름·회사·이메일 검색, 마지막 연락 시점까지 한 화면에. Phase 2에서 실 API CRUD로 전환 완료 — 추가·수정·삭제가 영속 저장되고 조직별 격리(RLS)로 다른 회사 고객은 보이지 않습니다.

✓ 현재 가능
  • 실 API CRUD — 추가/수정/삭제 모두 영속 저장 (`/customers` 6 endpoints)
  • 4 KPI (전체/활성/검토중/대기) — 조직 전체 기준 자동 갱신
  • 2 그룹 chip 필터 (status / sort) + 이름·회사·이메일 검색
  • URL query string 동기화 — 새로고침·뒤로 가기·북마크 복원
  • 듀얼 모드 모달 (신규 + 수정 + 삭제) · 24명 seed (Acme 12 / Beta 12)
  • 조직별 격리 — DB RLS로 다른 회사 고객 비노출, viewer 쓰기 403
  • 입력 검증 (zod 서버 + JSDoc 화면) · soft delete (보존)
  • 자동 회귀 — `phase_2_customers_e2e` 7 시나리오 + cleanup sweep
⌛ 아직 안 됨
  • 행 클릭 상세 패널 (통화 이력 결합) — Phase 4
  • 팀 단위 권한 분기 ("매니저는 자기 팀만") — Phase 3
  • CSV 일괄 가져오기 — Phase 6+ enterprise
  • HubSpot / Salesforce 양방향 동기화 — Phase 6 (v2 베타)
  • 고급 검색 (오타 허용·전문 검색) — Phase 5+
  • 삭제 고객 복원 UI — Phase 4+ 관리자 콘솔
07

뉴스레터 — AI 챗 기반 작성

"환영 메일 작성해줘" 한 줄로 개인화된 초안 자동 생성. 발송 후엔 전달률·오픈율·클릭률을 실시간 추적합니다.

✓ 현재 가능
  • 캠페인 5개 + 통계 mock
  • AI 챗봇 UI — 4종 프롬프트 (환영/업데이트/웨비나/갱신)별 다른 초안 자동 생성 + 타이핑 점 애니메이션
⌛ 아직 안 됨
  • 실제 LLM 연동 (Claude / GPT) — Phase 5
  • 실제 메일 발송 인프라 — Phase 3 (SMTP / SES)
  • 발송 통계 실시간 트래킹 — Phase 6
08

팀 & 계정 — 권한 매트릭스

관리자 1개 + 직원 서브 계정 (Pro 1명 / Enterprise 5명). 권한은 Admin · Manager · Employee · Viewer 4단계, 우수 사례 자동 노출로 팀 내 베스트 프랙티스를 즉시 공유.

✓ 현재 가능
  • 14명 구성원 + 우수 사례 랭킹 + 권한 매트릭스 UI
  • 직원 초대 모달 (mock 흐름)
  • 서버 측 role guard — admin / manager / employee / viewer 4단계 미들웨어 동작
⌛ 아직 안 됨
  • 실제 초대 메일 발송 — Phase 3 (SMTP)
  • 초대 토큰 검증 + 가입 완성 흐름 — Phase 3
  • 우수 사례 자동 추출 엔진 — Phase 5
09

설정 — 12 카테고리

프로필부터 위험 영역까지 12개 카테고리. 좌측 스크롤스파이 TOC가 현재 보고 있는 섹션을 항상 표시합니다.

✓ 현재 가능
  • 12 카테고리 — 프로필·회사·통화환경·AI·통합·알림·보안·데이터·플랜·API·언어·위험영역
  • 스크롤스파이 TOC — 보고 있는 섹션 자동 강조
⌛ 아직 안 됨
  • 각 항목 저장 — Phase 4+ (해당 기능 구현 시점에)
  • API 키 발급 — Phase 6
  • 2단계 인증 (TOTP) — Phase 3
10

Phase 1 + 2 종합 정리

지금 시점에서 무엇이 진짜로 동작하고 무엇이 mock인지를 한 표로 정리. Phase 1 기반(인증·격리·권한·세션·실시간) + Phase 2 첫 비즈니스 entity(고객 CRUD)가 모두 완료된 상태.

영역 현재 (Phase 1 + 2) 예정
데이터 영속 PostgreSQL 16 + RLS default-deny + 8 tables (customers 포함) Phase 4 — calls / transcripts CRUD
고객 관리 (CRM) 실 API CRUD 동작 — 6 endpoints + 4 KPI + 검색·필터·정렬 (URL 동기화) + soft delete + 24명 seed Phase 4 — 행 클릭 상세 패널 (통화 이력 결합)
인증 자체 Auth — Argon2id + JWT + refresh rotation + role guard Phase 3 — 회원가입/이메일 인증/비밀번호 재설정
조직별 격리 (RLS) DB 차원 강제 (Phase 1 인프라 + Phase 2 customers 정책 4 + 테스트로 회귀 보호) 새 entity 추가 시 같은 패턴
입력 검증 / 형식 동기화 zod 서버 + JSDoc 화면 + sync 테스트 (customer 형식 한 출처) Phase 3+ 새 entity도 동일 패턴
통화 어시스턴트 WebSocket (인증 핸드셰이크) + 데모 시퀀스 fixture Phase 5 — 실제 STT (Clova) + LLM (Claude) + RAG
대시보드 / KPI 고객 KPI는 실 데이터 (4종). 통화·매출 KPI는 mock Phase 4 — calls REST + 실 KPI 집계
시장 트렌드 5 포맷 다운로드 (HTML/PDF/Word/Excel/PPT) 실작동, 데이터는 mock Phase 4~5 — 네이버 검색 API + To-Do 엔진
팀 / 권한 role 미들웨어 4단계 동작 (customers는 viewer 차단까지 e2e 검증). 팀 단위 분기는 미도입 Phase 3 — 초대 메일 + 가입 완성 + 팀 단위 권한
자동 회귀 서버 단위 65/65 + sync 형식 + Phase 0.5 e2e 16/16 + customers e2e 7 시나리오 + cleanup sweep 새 entity마다 같은 패턴 추가
데스크톱 앱 웹 콘솔만 동작 별도 트랙 — Electron/Tauri + WASAPI 오디오 캡처
운영 / TLS localhost plaintext (dev). Caddy single-origin variant 검증 통과 Phase 6+ — 운영 도메인 / 결제 / 모니터링
