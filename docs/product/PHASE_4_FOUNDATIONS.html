PHASE 4 · PERSISTENT CALLS

통화가 끝나도 사라지지 않습니다 —
기록·집계·다시 보기가 진짜로 가능합니다

Phase 4에서는 다섯 가지가 새로 가능해집니다 — 통화 자체의 DB 영속화, 통화 기록 페이지의 실 API 전환, 대시보드의 실 데이터 KPI, 권한·인증 게이트, 자동 회귀로 만든 종료 게이트. 이제 평가자는 자기 손으로 통화를 만들고, 다시 열어 보고, 대시보드에서 집계까지 확인할 수 있습니다.

⌜ Phase 0.5는 통화 흐름을 실시간 WebSocket으로 연결했지만 데이터는 흘러간 뒤 사라졌습니다. Phase 4는 그 흐름을 DB로 묶고, 통화 기록과 대시보드를 진짜 데이터로 다시 켭니다. 이전 단계가 보장하던 회사별 분리·권한·입력 검증은 그대로 유지된 채, 이 새 흐름들이 그 위에 올라갑니다. ⌟

Phase 4의 약속

평가자가 직접 확인할 수 있는 5가지 기둥

1 · 통화 영속화

시작·발화·종료가 한 트랜잭션으로 DB에 남고, 고객의 마지막 연락 시각이 동시에 갱신됩니다.

2 · 통화 기록 실 API

정적 8건 데모는 사라지고, 검색·필터·상세 패널이 모두 본인 조직의 진짜 통화로 동작합니다.

3 · 실 데이터 대시보드

오늘 통화 / 응답률 / 평균 통화 / 진행 중 / 최근 5건이 단일 API 응답으로 채워집니다.

4 · 권한·인증 게이트

Viewer는 읽기만, 미인증 사용자는 변경 금지. 화면 위 배너로 즉시 보이고 서버가 강제합니다.

5 · 자동 회귀 종료 게이트

단위 212 + 4개 흐름 e2e가 매번 그린이어야만 Phase 4가 합격으로 인정됩니다.

기둥 01

통화 영속화 — 한 트랜잭션, 한 결과

통화가 시작되면 서버가 즉시 데이터베이스에 통화 행을 만듭니다. 통화 중에 흘러가는 발화는 같은 통화에 묶여 차곡차곡 적층되고, 종료 버튼은 통화 상태를 마감하면서 이 고객의 마지막 연락 시각까지 같은 트랜잭션에서 함께 갱신합니다. 통화 도중 회선이 끊겨도 이미 들어간 발화는 그대로 남고, 페이지를 새로고침해도 통화 기록에서 다시 볼 수 있습니다.

통화 한 건이 만드는 흐름

1

통화 시작 — WebSocket

화면이 서버와 통화 채널을 열고 시작 신호를 보냅니다. 서버는 곧바로 새 통화 행을 만들고 식별자를 돌려줍니다.

2

발화 적층 — 순서 보존

상담원과 고객의 말이 들어올 때마다 통화 식별자에 묶여 순번과 함께 저장됩니다. 같은 통화의 발화는 절대 섞이지 않습니다.

3

빠른 메모 — 즉시 저장

통화 중 입력한 한 줄 메모는 저장 버튼을 누르는 즉시 통화에 붙습니다. 통화 기록 화면의 메모 패널에 그대로 노출됩니다.

4

종료 — 고객 마지막 연락 갱신

종료 버튼은 통화 상태를 마감하면서, 이 통화의 고객 행에서 "마지막 연락 시각"도 같은 트랜잭션에서 갱신합니다.

한 트랜잭션 = 한 결과

종료 처리는 통화 상태 마감 + 고객의 마지막 연락 시각 갱신을 동시에 합니다. 중간에 실패하면 둘 다 없던 일이 되어, 통화는 끝난 것처럼 보이는데 고객 타임라인은 어긋난 상태가 절대 생기지 않습니다.

cross-org 참조 자체가 막힘

통화의 고객·상담원 참조는 모두 "조직 + 식별자" 두 키 조합으로 묶여 있어, 다른 회사 고객이나 상담원을 가리키는 통화는 데이터베이스가 처음부터 거절합니다.

부드러운 삭제 + 부분 색인

통화는 직접 지우는 대신 "삭제 표시"만 합니다. 모든 조회 색인은 삭제되지 않은 통화만 보도록 좁혀져 있어, 색인 크기와 쓰기 비용을 동시에 줄입니다.

발화 순번은 서버가 발급

한 통화의 발화 순번은 서버가 결정합니다. 두 발화가 같은 순번을 가질 가능성이 데이터베이스 차원에서 차단됩니다 (한 통화 안에서 순번은 유일).

기둥 02

통화 기록 페이지 — 정적 데모는 사라졌습니다

예전에는 통화 기록 페이지가 8건짜리 정적 예시를 화면에 그대로 박아 보여주는 데모였습니다. Phase 4에서 이 화면은 본인 조직의 진짜 통화를 검색·필터링·열람하는 도구로 바뀌었습니다. 브라우저 주소창의 검색·필터 상태가 그대로 URL에 살아남고, 행을 클릭하면 통화 본문·전사·다음 액션이 한 번에 열립니다.

이전 — Phase 3까지

  • · 화면 안에 직접 박혀 있는 8건짜리 예시 통화
  • · "총 1,243건" 같은 가짜 카운트
  • · 검색은 화면이 가진 8건만 substring 비교
  • · 행을 눌러도 같은 정적 예시만 펼쳐짐
  • · 다른 조직으로 로그인해도 같은 8건이 보임

Phase 4 — 지금

  • · 정적 배열 없음. 모두 본인 조직 데이터베이스 결과
  • · 헤더 카운트는 서버가 돌려준 진짜 총합
  • · 검색·상태 필터·페이지가 모두 서버 호출 (250ms 디바운스)
  • · 행을 누르면 통화 본문·전사·다음 액션을 동시에 조회
  • · 다른 회사 통화는 식별자 직격에도 존재 자체가 보이지 않음

URL이 곧 상태입니다

/platform/calls.html?q=합병&status=ended

이 URL을 그대로 북마크하거나 공유하면, 다음에 열었을 때 검색창이 자동으로 "합병"으로 채워지고 상태 필터가 "완료"에 고정된 상태로 진입합니다. 페이지 안에서 검색어를 바꾸면 URL이 즉시 같이 갱신됩니다 — 새로고침해도 같은 상태가 유지되는 것이 핵심입니다.

상세 패널 — 3개 동시 조회

행을 누르면 통화 본문·전사·다음 액션이 같은 시점에 요청되어 응답이 도착하는 대로 채워집니다. 한 번씩 순차 호출이 아니라 병렬이라 체감이 빠릅니다.

외부에서 흘러온 텍스트는 escape

통화 메모, 전사, 다음 액션 제목처럼 외부에서 흘러왔을 수 있는 텍스트는 화면에 박히기 전에 모두 안전한 형태로 변환됩니다. HTML이 섞여 들어와도 그대로 글자처럼만 표시됩니다.

CSV 내보내기 버튼은 아직 데모

우측 상단의 CSV 내보내기 버튼은 자리만 잡혀 있습니다. 실제 동작은 운영 단계(Phase 6+)에서 정해집니다.

기둥 03

대시보드 — 진짜 KPI 4장 + 최근 통화 5건

대시보드 첫 줄의 카드 네 장과 그 아래 최근 통화 목록은 모두 서버가 본인 조직의 데이터를 모아 한 번의 응답으로 돌려주는 결과입니다. 나머지 영역(시장 트렌드, 추천 To-Do, 팀 활동)은 아직 데모 상태이고 그 사실이 카드 헤더에 노란 라벨로 표시됩니다.

네 장의 카드와 최근 5건이 각각 따로 호출되지 않고, 단일 요약 API 한 번에 모두 채워집니다. 페이지 로드가 가볍고 일관성 있는 시점의 데이터입니다.

"오늘"은 UTC 자정 기준

현재 단계에서는 회사 단위 시간대 설정이 없어 서버 표준시(UTC) 자정 기준으로 "오늘"이 결정됩니다. 한국 시간 자정 직후 ~ 오전 9시 사이에는 *어제*로 분류된 통화가 일시적으로 섞일 수 있습니다 (운영 단계에서 회사별 시간대 도입 예정).

응답률의 정의

응답률 = 완료된 통화 ÷ (완료된 통화 + 미수신 통화). 네트워크 끊김으로 인한 통화는 분모에서 제외됩니다 — 응대 의지와 무관한 회선 사고이기 때문입니다.

최근 통화 5건은 LEFT JOIN

최근 통화 5건은 고객 이름·상담원 이름을 함께 보여주지만, 그 고객이나 상담원이 사라져도 통화 행 자체는 목록에서 빠지지 않습니다. "이름 없음"으로 표시되어 이력이 보존됩니다.

아직 데모인 영역

대시보드의 시장 트렌드, 추천 To-Do, 팀 활동은 아직 정적 예시입니다. 각 섹션 헤더에 노란 demo 라벨이 붙어 있어 평가 시점에 즉시 구분 가능합니다. 시장 트렌드는 운영 단계의 검색 API 연동 시점에, To-Do와 팀 활동은 AI 추천·활동 로그 도입 시점에 진짜 데이터로 바뀝니다.

기둥 04

권한과 인증 게이트 — 못 하는 일은 명확히

통화 데이터는 누구나 만질 수 있는 데이터가 아닙니다. 관리자·매니저·일반 직원·조회 권한이 각자 할 수 있는 일이 다르고, 이메일 인증을 마치지 않은 사용자는 화면 위에 노란 띠로 안내가 뜨면서 통화 변경이 모두 거절됩니다.

"본인 통화"란 통화 행에 박혀 있는 상담원 식별자가 지금 로그인한 사용자와 같은 경우를 말합니다. 매니저가 자기 팀 통화만 다루는 더 세밀한 정책은 다음 단계(Phase 5)에서 매니저 보고서와 함께 도입됩니다.

미인증 사용자가 보호 화면에 들어가면

이메일 인증이 완료되지 않았습니다. 통화 변경은 인증 후에 가능합니다.

상단의 노란 띠가 미인증 상태를 화면에 즉시 알립니다. 통화 기록·대시보드 두 페이지 모두 이 띠가 켜진 상태에서 진입합니다. 그 위에 서버가 통화 변경 시도를 모두 거절합니다 — 읽기는 여전히 가능하기 때문에, 인증을 마치지 않아도 자기 조직 통화를 둘러볼 수는 있습니다.

남의 통화를 만지려고 하면

일반 직원이 다른 사람 통화의 메모를 바꾸려고 하면 즉시 "권한 없음"으로 거절됩니다. 같은 조직 안에서도 본인 통화 외에는 손댈 수 없습니다 (조회만 가능).

다른 회사 통화는 식별자도 노출되지 않음

"다른 회사 통화의 ID를 안다고 가정"하고 그 통화를 조회·변경하려고 하면, 서버는 "그런 통화 없음"으로 응답합니다. 존재 자체가 드러나지 않는 것이 핵심입니다.

로그아웃 후 다시 들어오면 같은 약속

로그인 세션이 끊겨도, 다시 로그인한 뒤 위 규칙들이 그대로 적용됩니다. 새로 발급된 토큰의 역할·인증 상태를 매번 다시 확인합니다.

역할이 바뀐 직후의 안전망

방금 관리자에서 일반 직원으로 강등된 사용자가 옛 토큰으로 변경을 시도하면, 서버가 데이터베이스의 현재 역할을 다시 읽어 차이를 잡아내고 재로그인을 요구합니다.

Phase 5 인계 — 진짜 음성과 AI

Phase 4가 통화 영속화의 토대를 깔았다면, Phase 5에서는 그 위에 실제 음성 인식과 AI 분석이 올라갑니다. 통화 화면의 데모 영역이 진짜 데이터로 바뀌고, 매니저용 팀 단위 시야가 도입됩니다.

실제 음성 인식 (STT)

Phase 0.5의 데모 발화 흐름이 실제 마이크 입력을 한국어로 변환하는 외부 STT 어댑터로 교체됩니다. 영속화 흐름은 그대로 살아 있어 새 음성이 들어오면 즉시 통화에 적층됩니다.

AI 요약·응대 추천 자동 생성

통화 후 요약·미해결 이슈·다음 액션이 회사 가이드 기반 AI로 자동 생성되어 통화에 붙습니다. 통화 중에는 실시간 응대 추천 카드가 진짜 LLM 응답으로 갱신됩니다.

상담 체크리스트·빠른 멘트 영속화

지금은 화면에 박혀 있는 5항목 체크리스트와 빠른 응대 멘트가 회사별 설정 데이터로 옮겨가고, 통화 화면에서 토글한 결과가 통화 기록에 함께 저장됩니다.

통화 회선 끊김 자동 처리

브라우저 비정상 종료로 인해 통화가 "진행 중"으로 남는 케이스를 서버가 자동으로 "끊김"으로 마감합니다. 진행 중 KPI가 자연스럽게 정상화됩니다.

고객 선택 흐름

통화 시작 시점에 고객을 골라 통화에 연결할 수 있는 흐름이 추가되어, 좌측 고객 카드가 진짜 고객 데이터로 채워집니다. 통화 후에는 사후 매칭도 가능합니다.

매니저 팀 시야

"매니저는 자기 팀 통화만"이라는 더 좁은 권한 정책이 도입됩니다. 지금은 매니저가 자기 회사 전체 통화를 볼 수 있는 상태에서 출발합니다.

신규 스키마 4 migration

calls · transcripts · call_action_items 테이블 + RLS FORCE + 부분 인덱스 4쌍 + 부속 app role grants. 모두 forward-only.

composite FK guard

calls(org_id, customer_id) → customers(org_id, id) + calls(org_id, agent_user_id) → memberships(org_id, user_id). cross-org 참조 자체를 PostgreSQL이 거절.

endCall 트랜잭션

calls 상태/종료시각/지속시간 UPDATE + customer_id가 있으면 customers.last_contacted_at = GREATEST(기존, ended_at)을 같은 트랜잭션. trigger 사용 안 함.

requireVerified 위치

requireAuth → orgContext → requireVerified → requireRole → requireFreshRole. 비활성 멤버는 같은 hook에서 401 stale_session으로 통합 분기.

WebSocket persistence hook

start_call 시 calls insert, text_chunk 시 transcripts append + echo, end_call 시 service.endCall. agent_user_id는 server-side에서 socket.user.id로 고정.

shared types 9 entity

server/src/types/* zod 원본 + platform/types/*.js JSDoc 사본 + test/sync_shared_types.mjs. Phase 4에서 call / transcript / actionItem / dashboard 4 entity 추가.

dashboard "오늘" 정의

date_trunc('day', now() AT TIME ZONE 'UTC') 기준. 조직 단위 timezone 도입은 Phase 6+.

innerHTML XSS gate

calls.html / dashboard.html server-supplied 보간 자리에 escapeHtml 명시. live.html transcript는 textContent, AI suggestion은 DOMPurify.

e2e cleanup 범위

phase4test- prefix가 붙은 통화·발화·다음 액션 + phase4test-%@example.test 사용자가 만든 통화만 hard-delete. 무관한 수동 통화는 손대지 않음.
