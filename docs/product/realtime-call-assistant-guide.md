# 실시간 통화 AI 어시스턴트 구축 가이드

## 1. 제품 정의

이 문서는 고객사 PC에 설치되는 데스크톱 앱을 통해 상담 통화 내용을 실시간으로 수집하고, 우리 서버에서 AI 분석을 수행해 상담원에게 보조 정보를 제공하는 시스템의 구축 방향을 정의한다.

목표는 전화망을 직접 대체하는 AI 콜센터가 아니라, 상담원이 고객과 통화하는 동안 AI가 실시간으로 다음 정보를 제공하는 **상담원 보조형 AI 어시스턴트**다.

- 실시간 대화록
- 고객 발화 요약
- 상담 흐름 감지
- 추천 답변
- 체크리스트
- 이슈/니즈/반박 포인트 추출
- 통화 종료 후 상담 메모 자동 생성

기준 실시간성은 **2~3초 지연 허용**으로 잡는다. 이 수준이면 완전한 동시통역형 응답은 아니지만, 상담원이 대화를 이어가며 참고하는 어시스턴트 용도로는 현실적이다.

## 2. 핵심 전제

고객사 환경을 모두 지원하지 않는다. 고객사는 우리 시스템이 요구하는 표준 환경에 맞춰야 하며, 해당 조건을 수용하는 고객사만 온보딩한다.

표준 고객 환경은 다음과 같이 정의한다.

- 상담원은 Windows PC를 사용한다.
- 상담 통화는 PC 기반 소프트폰 또는 PC에서 오디오 캡처 가능한 환경에서 진행한다.
- 상담원은 우리 데스크톱 앱을 설치하고 로그인한다.
- 앱은 마이크 입력과 통화 상대방 음성을 캡처할 권한을 가진다.
- 통화 데이터가 우리 서버로 전송되는 것에 대해 고객사와 필요한 계약, 고지, 동의 절차를 완료한다.
- 고객사는 상담 스크립트, 상품 정보, FAQ, 금지 표현, 업무 프로세스를 사전에 등록한다.

초기 버전에서는 일반 유선전화, 별도 IP폰, 폐쇄형 PBX, 녹취 장비 직접 연동은 지원 범위에서 제외한다. 필요 시 별도 엔터프라이즈 연동 프로젝트로 분리한다.

## 3. 전체 아키텍처

```text
[상담원 PC 앱]
  - 마이크 오디오 캡처
  - 시스템/통화 오디오 캡처
  - 음성 버퍼링
  - 로컬 VAD
  - WebSocket/WebRTC 업로드
        |
        v
[실시간 스트리밍 서버]
  - 세션 인증
  - 오디오 스트림 수신
  - 상담원/고객 채널 관리
  - STT 파이프라인 연결
  - 이벤트 브로커
        |
        v
[AI 처리 서버]
  - 실시간 전사
  - 발화자 분리
  - 문맥 요약
  - 추천 답변 생성
  - 상담 가이드 매칭
  - 리스크/금지 표현 감지
        |
        v
[상담원 콘솔]
  - 실시간 대화록
  - AI 추천 답변
  - 상담 체크리스트
  - 고객 상태/니즈 표시
  - 통화 후 메모 저장
```

## 4. PC 앱 역할

PC 앱은 AI 판단을 직접 수행하지 않고, 안정적인 오디오 수집과 서버 전송에 집중한다.

필수 기능은 다음과 같다.

- 사용자 로그인 및 디바이스 등록
- 마이크 입력 캡처
- 시스템 오디오 또는 소프트폰 출력 캡처
- 통화 시작/종료 감지 또는 수동 시작/종료 버튼
- 오디오 청크 단위 스트리밍
- 네트워크 끊김 시 짧은 로컬 버퍼링
- 전송 상태 표시
- 앱 자동 업데이트
- 로그 수집 및 장애 리포트

권장 구현은 다음과 같다.

- Windows 앱: Electron, Tauri, 또는 .NET/WPF
- 오디오 캡처: WASAPI loopback + microphone capture
- 전송 방식: WebSocket 우선, 저지연 양방향 제어가 필요하면 WebRTC 검토
- 오디오 포맷: 16kHz 또는 24kHz mono PCM/Opus
- 청크 크기: 100~300ms 단위

초기 MVP에서는 Electron 또는 Tauri가 빠르다. 단, 오디오 캡처 안정성과 배포/업데이트까지 고려하면 Windows 네이티브 모듈 연동이 필요할 수 있다.

## 5. 서버 역할

서버는 통화 세션을 관리하고, 음성 데이터를 AI 파이프라인으로 넘기며, 결과를 상담원 화면에 실시간으로 반환한다.

필수 구성요소는 다음과 같다.

- 인증 서버
- 조직/상담원/디바이스 관리
- 실시간 스트리밍 게이트웨이
- STT 처리 워커
- AI 어시스턴트 워커
- 상담 지식베이스 검색
- 통화 로그 저장소
- 관리자 콘솔 API

세션 단위 데이터 모델은 다음 개념을 가진다.

- organization
- agent
- device
- call_session
- audio_stream
- transcript_segment
- ai_suggestion
- call_summary
- follow_up_task

## 6. 실시간 처리 파이프라인

2~3초 지연을 목표로 하는 파이프라인은 다음 흐름이 적합하다.

```text
오디오 캡처
  -> 100~300ms 청크 생성
  -> 서버 전송
  -> VAD/무음 제거
  -> STT
  -> 발화 단위 세그먼트 생성
  -> 최근 문맥 윈도우 업데이트
  -> AI 분석
  -> 상담원 콘솔 업데이트
```

AI 분석은 매 청크마다 수행하지 않는다. 비용과 안정성을 위해 발화 단위 또는 1~2초 단위로 묶어 처리한다.

추천 답변 생성 기준은 다음과 같이 나눈다.

- 짧은 응답 추천: 최근 고객 발화 1~3개 기준
- 상담 흐름 판단: 최근 1~2분 대화 기준
- 통화 요약: 전체 누적 대화 기준
- 후속 액션: 통화 종료 시 전체 대화 기준

## 7. AI 기능 범위

초기 버전에서 제공할 기능은 다음 순서가 현실적이다.

1. 실시간 전사
2. 통화 중 핵심 키워드 추출
3. 상담 스크립트/FAQ 매칭
4. 추천 답변 표시
5. 고객 니즈/불만/반박 포인트 감지
6. 통화 종료 후 자동 요약
7. CRM 메모 자동 생성

고급 기능은 MVP 이후로 분리한다.

- 상담원 말투 코칭
- 실시간 컴플라이언스 경고
- 감정 분석
- 고객 구매 가능성 스코어링
- 자동 follow-up 메시지 생성
- 관리자용 팀 성과 분석

## 8. 상담원 콘솔 UX

상담원 화면은 통화 중 빠르게 읽을 수 있어야 한다. 긴 설명형 UI보다 짧은 카드와 상태 표시가 적합하다.

권장 화면 구성은 다음과 같다.

- 좌측: 실시간 대화록
- 중앙: 현재 고객 의도와 추천 답변
- 우측: 상담 체크리스트, 고객 정보, 금지/주의 표현
- 하단: 통화 상태, 전송 상태, 메모 저장 버튼

추천 답변은 상담원이 그대로 읽는 스크립트가 아니라, 말할 방향을 빠르게 잡는 보조 문장으로 제공한다.

예시:

```text
고객 의도: 가격 부담
추천 방향: 비용보다 회수 기간과 리스크 감소를 먼저 설명
추천 멘트: "월 비용만 보면 부담이 될 수 있지만, 현재 놓치는 상담 기록과 후속 액션을 줄이는 쪽에서 먼저 효과를 보실 수 있습니다."
```

## 9. 지식베이스

AI가 아무 근거 없이 답변하지 않도록 고객사별 지식베이스를 둔다.

필수 등록 데이터는 다음과 같다.

- 회사 소개
- 상품/서비스 설명
- 가격 정책
- FAQ
- 상담 스크립트
- 반박 처리 가이드
- 금지 표현
- 개인정보 처리 기준
- 상담 후속 액션 규칙

검색 방식은 RAG 구조를 사용한다.

```text
현재 대화 문맥
  -> 고객사 지식베이스 검색
  -> 관련 문서 조각 추출
  -> AI 추천 답변 생성
```

고객사별 데이터는 반드시 tenant 단위로 분리한다.

## 10. 보안과 개인정보

통화 내용은 민감정보가 포함될 수 있으므로 초기 설계부터 보안 요구사항을 제품 요건으로 포함한다.

필수 요구사항은 다음과 같다.

- 전송 구간 TLS 암호화
- 조직별 데이터 분리
- 상담원별 접근 권한
- 관리자 감사 로그
- 녹취/전사 저장 여부 설정
- 보관 기간 설정
- 삭제 요청 처리
- 민감정보 마스킹
- 운영자 접근 통제

초기 MVP에서도 최소한 다음은 반드시 구현한다.

- 로그인 기반 인증
- 조직별 데이터 분리
- 통화 세션 접근 권한 확인
- 전송 암호화
- 저장 데이터 삭제 기능

## 11. MVP 범위

1차 MVP는 다음 범위로 잡는다.

- Windows PC 앱
- 마이크 + 시스템 오디오 캡처
- WebSocket 실시간 업로드
- 서버 STT
- 실시간 대화록 표시
- 고객사 FAQ 기반 추천 답변
- 통화 종료 후 자동 요약
- 관리자 페이지에서 FAQ/스크립트 등록

제외 범위는 다음과 같다.

- 모든 전화 환경 자동 지원
- PBX/CTI 연동
- 상담원 자동 배정
- AI가 직접 고객과 통화
- 결제/청구 자동화
- 대규모 통계 대시보드
- 완전한 CRM 제품화

## 12. 기술 선택안

권장 초기 스택은 다음과 같다.

- PC 앱: Electron 또는 Tauri
- 백엔드: Node.js/NestJS 또는 FastAPI
- 실시간 통신: WebSocket
- 비동기 처리: Redis Streams 또는 NATS
- DB: PostgreSQL
- 파일/원본 오디오 저장: S3 호환 스토리지
- 벡터 검색: pgvector 또는 전용 벡터 DB
- AI: 실시간 STT + LLM/RAG
- 관리자/상담원 웹: React

OpenAI Realtime API를 사용할 경우, 음성-음성 대화뿐 아니라 실시간 오디오 입력과 이벤트 기반 제어를 활용할 수 있다. 다만 이 제품은 상담원 보조가 목적이므로 초기에는 “STT + 텍스트 기반 AI 분석” 구조가 디버깅과 비용 통제에 유리하다.

## 13. 주요 리스크

가장 큰 리스크는 모델 성능보다 오디오 수집 안정성이다.

- 고객 PC마다 오디오 장치 구성이 다를 수 있다.
- 소프트폰 앱이 오디오 캡처를 제한할 수 있다.
- 마이크와 시스템 오디오가 섞이면 발화자 구분이 어려워진다.
- 네트워크 품질이 나쁘면 실시간성이 무너진다.
- AI 추천이 너무 자주 나오면 상담원에게 방해가 된다.
- 잘못된 추천 답변은 고객 응대 품질을 떨어뜨릴 수 있다.

따라서 초기 파일럿은 표준 장비와 표준 소프트폰을 지정해서 진행한다.

권장 파일럿 조건:

- 상담원 3~5명
- 동일한 헤드셋
- 동일한 소프트폰
- 동일한 Windows 버전
- 하루 20~50콜 수준
- 저장/전송 동의 절차 완료

## 14. 개발 단계

### 1단계: 로컬 캡처 검증

- Windows에서 마이크와 시스템 오디오를 동시에 캡처한다.
- 10분 이상 끊김 없이 녹음되는지 확인한다.
- 상담원/고객 음성이 모두 들어오는지 확인한다.

### 2단계: 스트리밍 검증

- PC 앱에서 서버로 오디오 청크를 전송한다.
- 서버에서 청크를 조립하거나 실시간 STT로 넘긴다.
- 평균 지연시간을 측정한다.

### 3단계: 실시간 전사

- 통화 중 전사 결과를 웹 콘솔에 표시한다.
- 무음/중복/끊김 처리를 개선한다.
- 발화 단위 세그먼트를 만든다.

### 4단계: AI 추천

- 고객사 FAQ와 상담 스크립트를 등록한다.
- 최근 대화 문맥을 기준으로 추천 답변을 생성한다.
- 추천 빈도와 표시 방식을 조정한다.

### 5단계: 통화 후 자동 메모

- 전체 전사 내용을 요약한다.
- 고객 니즈, 문제, 다음 액션을 추출한다.
- 상담원이 수정 후 저장할 수 있게 한다.

## 15. 성공 기준

MVP 성공 기준은 다음과 같이 정의한다.

- 통화 중 전사 지연이 평균 3초 이내다.
- 30분 통화에서도 앱이 안정적으로 동작한다.
- 상담원이 추천 답변을 방해가 아니라 도움으로 인식한다.
- 통화 후 메모 작성 시간이 50% 이상 줄어든다.
- 고객사 FAQ 기반 답변이 명백히 틀린 정보를 만들지 않는다.
- 관리자 없이도 고객사 스크립트를 등록하고 갱신할 수 있다.

## 16. 결론

이 제품은 고객사가 우리 표준 환경에 맞춘다는 전제에서는 충분히 현실적이다. 초기에는 전화망/PBX 연동을 피하고, PC 앱 기반 오디오 캡처와 실시간 STT/AI 분석에 집중해야 한다.

가장 먼저 검증할 것은 AI 답변 품질이 아니라 **고객 PC에서 통화 양쪽 음성을 안정적으로 캡처하고 2~3초 안에 서버 분석 결과를 돌려줄 수 있는지**다. 이 검증이 통과되면 상담원 보조형 AI SaaS로 제품화할 수 있다.
