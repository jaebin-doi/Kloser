PHASE 1 · FOUNDATIONS

Kloser를 안전하게 쓸 수 있는
6가지 기반

통화 어시스턴트, 자동 메모, AI To-Do — Kloser의 핵심 업무 기능을 본격적으로 올리기 전에, 계정·조직·권한 단위로 흔들림 없이 사용할 수 있는 기반이 먼저 필요합니다. Phase 1은 그 기반을 만든 단계입니다.

⌜ Phase 1은 최종 업무 기능을 완성한 단계가 아니라, Kloser를 계정·조직·권한 단위로 안전하게 사용할 수 있게 만드는 기반 단계입니다. ⌟

기반의 구성

사용자가 신뢰할 수 있는 6가지 기둥

1 · 로그인

회사 계정으로 들어오면 사용자·조직·권한이 함께 결합됩니다.

2 · 조직 분리

자기 조직 데이터만 보입니다. 다른 조직은 어떤 방식으로도 접근 불가.

3 · 권한

Admin / Manager / Employee / Viewer 4단계로 가능한 작업이 달라집니다.

4 · 세션 유지

15분 접속 권한 + 30일 로그인 유지. 로그아웃하면 즉시 종료.

5 · 인증된 실시간 연결

통화 어시스턴트가 쓰는 실시간 채널도 인증된 사용자만.

6 · 데이터 보호

비밀번호는 원문 미저장. 운영/일상 권한 분리. 의심 활동 시 자동 무효화.

기둥 01

로그인 — 사용자·조직·권한이 한 번에 결합됩니다

회사가 발급한 이메일과 비밀번호로 들어오면, 그 사용자가 누구인지, 어느 조직에 속해 있는지, 어떤 권한을 가졌는지가 한 번에 묶여 모든 화면과 기능에 적용됩니다. 이후 URL을 바꾸거나 요청을 직접 조작해도 이 결합된 신원을 넘어서는 일은 일어나지 않습니다.

사용자 입력

user@kloser.com

결합된 신원

  • ✓ 사용자 김민수
  • ✓ 소속 조직 Kloser Inc.
  • ✓ 권한 EMPLOYEE

비밀번호는 원문으로 저장되지 않습니다

한 방향으로만 변환된 형태로 보관됩니다. 시스템 관리자도 사용자의 원래 비밀번호를 알 수 없습니다.

여러 조직에 속한 경우

로그인 시점에 어느 조직으로 들어갈지 명시합니다. 한 번 결정된 조직은 로그아웃 전까지 바뀌지 않습니다.

기둥 02

조직 분리 — 다른 회사 데이터는 보이지 않습니다

Kloser는 회사 단위 SaaS입니다. 한 화면에 여러 회사의 데이터가 섞여 보이는 일은 절대 없어야 합니다. 분리는 한 곳이 아니라 두 곳에서 검증됩니다 — 서버 측에서 사용자의 조직을 확인하고, 데이터베이스에서 한 번 더 차단합니다.

조직 A — Acme Corp

사용자가 보고 있는 화면

  • 고객 12명
  • 통화 기록 8건
  • 팀원 14명
서버 + DB 양쪽 차단

URL이나 요청값을 바꿔도 통과하지 않습니다.

조직 B — Beta LLC

사용자에게 절대 보이지 않는 데이터

  • 고객 (보이지 않음)
  • 통화 기록 (보이지 않음)
  • 팀원 (보이지 않음)

왜 이중 검증인가: 한쪽 코드에 실수가 있더라도 다른 쪽이 막아줍니다. 평가 시점에 "URL을 바꿔서 다른 회사 데이터를 볼 수 있나요?" 같은 시도는 모두 거부됩니다.

기둥 03

권한 — 4단계로 가능한 작업이 달라집니다

같은 조직 안에서도 누구나 똑같은 일을 할 수 있는 건 아닙니다. 권한이 더 높다고 해서 모든 걸 할 수 있는 것도 아닙니다 — 예를 들어 "조회" 권한은 화면이 동일하게 보여도 변경·삭제 요청은 시스템 차원에서 거부됩니다. 화면 비활성화에만 의존하지 않습니다.

권한 변경 반영: admin이 누군가의 권한을 바꿔도 그 사람의 현재 접속 권한은 즉시 무효화되지 않습니다. 다음 갱신 시점에 새 권한이 반영됩니다. 접속 권한은 기본 15분 동안만 유효하므로 최대 15분 안에 새 권한으로 바뀝니다. 파괴적 작업(예: 결제 변경, 대량 삭제)은 그 시점에 한 번 더 권한을 재확인합니다.

기둥 04

세션 유지 — 안전과 편의가 분리됩니다

한 번 로그인하면 매번 다시 입력하지 않아도 됩니다. 기본 설정에서는 30일 동안 세션을 유지할 수 있고, 그 안에서 실제 요청 권한은 15분 단위로 짧게 발급됩니다. 이 "이어짐"은 두 가지로 분리돼 있습니다.

15분

짧은 접속 권한

실제 요청마다 함께 보내는 짧은 접속 권한입니다. 기본 유효 시간은 15분입니다. 브라우저 메모리에만 보관되며, 만료되면 어떤 요청도 통과시키지 않습니다.

30일

안전한 갱신 절차

접속 권한이 만료됐을 때 새 접속 권한을 받기 위한 로그인 유지 정보입니다. 기본 유효 시간은 30일이며, 일반 화면 코드가 직접 읽을 수 없는 안전한 쿠키로 보관됩니다.

세션 수명 (예시)

15분짜리 접속 권한은 필요할 때 다시 발급되고, 로그인 유지 정보는 기본 30일 동안 유지됩니다.

로그인 15분 후
필요 시 갱신 15분 후
필요 시 갱신 30일 후
다시 로그인

자동 유지

접속 권한이 만료되면 자동으로 새 권한을 받은 뒤 원래 요청을 한 번 다시 시도합니다.

로그아웃 시 즉시 종료

로그아웃한 세션 정보로는 어떤 시도도 통과하지 않습니다.

의심 활동 감지 시 차단

교체된 로그인 유지 정보가 30초 예외 시간을 지나 다시 쓰이면 관련 세션 전체가 무효화됩니다.

기둥 05

인증된 실시간 연결 — 통화 채널도 검증을 통과해야 합니다

통화 어시스턴트(전사·추천·감정 분석 화면)는 브라우저와 서버가 실시간으로 메시지를 주고받는 연결을 사용합니다. 이 연결도 일반 화면 호출과 동일한 인증 절차를 거칩니다. 인증되지 않은 연결은 그 자체가 성립하지 않습니다.

정상 흐름

STEP 1

연결 시작

로그인된 사용자 정보를 함께 보냅니다.

STEP 2 ✓

서버 검증 통과

사용자·조직·권한이 연결 단위에 결합됩니다.

STEP 3

메시지 흐름 시작

전사·추천·감정 데이터가 실시간으로 도착합니다.

차단되는 경우

STEP 1

연결 시도

권한 정보가 없거나 만료된 상태.

STEP 2 ✗

검증 실패

인증 코드 (`missing_token` / `expired_token` / `invalid_token`)와 함께 거부.

STEP 3

연결 자체가 안 열림

접속 권한이 만료되면 새 권한 발급을 한 번 시도합니다. 실패하면 로그인 화면으로 이동합니다.

연결이 살아있는 동안 사용자의 조직 정보가 모든 메시지에 함께 묶입니다. 통화 데이터 역시 조직 분리 규칙을 그대로 따릅니다.

기둥 06

데이터 보호 — 여러 겹의 안전장치

Phase 1 시점에 저장되는 핵심 데이터(회사·사용자·소속·세션)는 다음 4가지 층의 보호를 동시에 받습니다. 한 층이 뚫려도 다음 층이 막아냅니다.

1

비밀번호는 한 방향으로 변환되어 저장

원문 비밀번호는 어디에도 남지 않습니다. 비교가 필요할 때는 같은 변환 절차를 거친 결과끼리 비교합니다.

2

세션 정보는 안전한 형태로 보관 + 자연스러운 폐기

평문 형태로 저장되지 않습니다. 만료 시간이 함께 박혀 있어 시간이 지나면 자동으로 무효가 됩니다.

3

운영 권한과 일상 권한이 분리

일상 요청을 처리하는 쪽은 다른 조직 데이터를 들여다볼 수 없도록 권한 자체를 제한받습니다. 운영용 권한은 별도이며 일상 코드 경로에 노출되지 않습니다.

4

의심 활동 시 관련 세션 전체 즉시 무효화

교체된 로그인 유지 정보가 30초 예외 시간을 지나 다시 쓰이면 탈취 가능성으로 보고, 관련 세션 전체가 즉시 무효화됩니다.

SCOPE

Phase 1이 다루는 것 / 다루지 않는 것

Phase 1은 기반 단계입니다. 여기서 만든 안전한 기반 위에, 다음 Phase부터 본격적인 업무 기능이 올라갑니다.
